## 1. 产品概述

### 1.1 产品定位
面向中小型服务企业（清洁公司、物业管理、家政公司）的智能排班系统，帮助管理者用 10 分钟完成原本 2 小时的排班工作，减少人力浪费和客户投诉。

### 1.2 目标用户（含画像）

| 用户角色 | 画像 | 核心痛点 |
|---------|------|---------|
| **管理者** | 中小企业老板/运营经理，30-50岁，技术能力一般 | 每天花 2 小时手动排班，容易出错，员工抱怨不公平 |
| **调度员** | 专职调度，25-40岁，需要同时管理 20-50 名员工 | 电话/微信来回确认，信息混乱，临时调班手忙脚乱 |
| **一线员工** | 清洁工/技术员，25-55岁，主要用手机 | 不知道自己下一天的排班，临时通知容易错过 |

### 1.3 核心价值（3个核心问题+解决方案）

| 问题 | 现状 | 解决方案 |
|------|------|---------|
| 排班耗时 | 手动 Excel 排班，每次 1-2 小时 | AI 自动排班，10 分钟出结果 |
| 临时调班混乱 | 微信群通知，容易遗漏 | 系统自动通知 + 一键换班 |
| 员工不公平感 | 管理者主观分配 | 透明规则 + 自动轮换 |

### 1.4 成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|---------|
| 排班时间 | ≤ 15 分钟/周 | 管理者自评 |
| 员工满意度 | ≥ 4.0/5 | 月度问卷 |
| 排班冲突率 | ≤ 2% | 系统统计 |
| 临时调班处理时间 | ≤ 30 分钟 | 系统记录 |

## 2. 功能需求

### 2.1 员工管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 员工档案 | 姓名、电话、技能标签、可用时间段 | P0 |
| 技能标签 | 支持多标签（如：清洁、灭虫、高空作业） | P0 |
| 可用时间 | 每周设置可上班时间段 | P0 |
| 请假管理 | 员工提交请假，管理者审批 | P1 |

### 2.2 智能排班

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 自动排班 | 根据需求+技能+可用时间自动生成排班表 | P0 |
| 手动调整 | 管理者可拖拽调整排班 | P0 |
| 冲突检测 | 自动检测时间冲突、技能不匹配 | P0 |
| 排班模板 | 保存常用排班模式（如：周末班、节假日班） | P1 |

### 2.3 任务管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 任务创建 | 客户名称、地址、服务类型、时间要求 | P0 |
| 任务分配 | 自动/手动分配给合适员工 | P0 |
| 任务状态 | 待开始→进行中→已完成→客户确认 | P0 |
| 地图展示 | 在地图上展示当日任务分布 | P2 |

### 2.4 通知系统

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 排班通知 | 自动推送排班结果给员工（微信/短信） | P0 |
| 调班通知 | 临时变更自动通知相关方 | P0 |
| 提醒通知 | 上班前 1 小时提醒 | P1 |

### 2.5 数据报表

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 工时统计 | 每人每周/月工时汇总 | P1 |
| 出勤率 | 出勤/迟到/早退统计 | P1 |
| 人效分析 | 每人每日完成任务数 | P2 |

## 3. 非功能需求

## 4. 用户故事地图

```
员工注册 → 设置可用时间 → 查看排班 → 接受任务 → 完成打卡 → 查看工资
    ↓           ↓              ↓          ↓           ↓
管理者：添加员工 → 创建排班 → 分配任务 → 确认完成 → 导出报表
```
